◆ PPIDD_Create()

int PPIDD_Create ( char *  name,
int64_t  lentot,
int  dtype,
int  storetype,
int *  handle 
)

Create an array using the regular distribution model and return integer handle representing the array.

  • dtype=0 : Fortran integer and logical types
  • dtype=1 : Fortran double precision type
  • storetype=0 : Normal distributed array stored across the distributed processes
  • storetype>=1: Low-latency array stored on one or more helper processes (effective only when helper process is enabled). storetype is advisory: the underlying implementation will use up to storetype helpers.
  • For GA, storetype doesn't take effect, and data are always stored across the distributed processes, analogous to http://hpc.pnl.gov/globalarrays/api/c_op_api.html#CREATE
  • For MPI2, the library can presently be built with zero or one (default) helpers. When helper process is disabled, storetype doesn't take effect, and data are always stored across the distributed processes.

◆ PPIDD_Create_mutexes()

int PPIDD_Create_mutexes ( int  storetype,
int  number 
)

Create a set of mutex variables that can be used for mutual exclusion.

Returns .true. if the operation succeeded or .false. when failed. It is a collective operation.

  • For GA, storetype doesn't take effect, analogous to http://hpc.pnl.gov/globalarrays/api/c_op_api.html#CREATE_MUTEXES

  • For MPI2, there are two kinds of mutexes:

    • (1) storetype=0: mutex data will be stored by a global array across the distributed processes. Due to the poor performance of MPI-2 one-sided communication, this kind of mutex is very slow.
    • (2) storetype=1: mutex data will be stored on the helper process. Improve the performance of (1), and all communications are based on two-sided opeartions.

    When helper process is disabled, storetype doesn't take effect, and mutex data are always stored across the distributed processes.
